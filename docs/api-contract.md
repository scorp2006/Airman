# API Contract

The canonical contract lives at **http://127.0.0.1:8000/docs** (OpenAPI auto-generated by FastAPI). This file is a human-readable summary.

## Conventions

- All endpoints (except `/auth/login` and `/`) require the header `X-User-Id: <id>` — the mock authentication.
- Errors return `{ "detail": "human readable message" }` with appropriate HTTP status:
  - **400** invalid request (bad state transition, validation, business rule)
  - **401** missing/invalid X-User-Id header
  - **403** role not allowed for this action
  - **404** entity not found
  - **422** Pydantic validation failed (malformed body)

```http
→ 200 OK { ..., "status": "RELEASED" }

PATCH /sorties/6/release
X-User-Id: 11                                 # cadet
→ 403 Forbidden  { "detail": "Role CADET is not allowed for this action" }

PATCH /sorties/6/airborne                     # tries to skip
X-User-Id: 8
(after sortie is in SCHEDULED, not RELEASED)
→ 400 Bad Request { "detail": "Invalid state transition: SCHEDULED -> AIRBORNE" }
```

## Training

```http
POST /training-progress
X-User-Id: 9    (instructor)
{
  "sortie_id": 6,
  "maneuver_score": 4,
  "communication_score": 3,
  "situational_awareness_score": 4,
  "remarks": "Good handling, work on radio calls"
}
→ 201 Created { status: "SUBMITTED", ... }

PATCH /training-progress/1/approve
X-User-Id: 10   (CFI)
→ 200 OK { status: "APPROVED" }

PATCH /training-progress/1/reject
X-User-Id: 10   (CFI)
{ "remarks": "Scores too generous given communication issues" }
→ 200 OK { status: "REJECTED" }   # sortie goes back to LANDED
```

## Audit logs

```http
GET /audit-logs?entity_type=Sortie&action=SORTIE_RELEASED
X-User-Id: 1
→ 200 OK [ { actor_id, action, entity_type, entity_id, old_value, new_value, created_at }, ... ]
```

Filters: `entity_type`, `action`, `actor_id`, `entity_id`. All optional. Limit 500 rows.
